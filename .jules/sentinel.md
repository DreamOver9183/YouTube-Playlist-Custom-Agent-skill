## 2024-06-15 - [Critical] Path Traversal in yt_tool.py via playlist_id
**Vulnerability:** A path traversal vulnerability existed in `scripts/yt_tool.py` when handling playlist IDs. The `extract_id` function returned the raw input if it didn't match a YouTube URL regex. This user-controlled input was then used directly in constructing file paths (e.g., `progress_file = LOG_DIR / f"progress_{playlist_id}.json"`), allowing an attacker to write files outside the intended `logs/` directory using sequences like `../../../`.
**Learning:** Even when inputs seem to be meant for external APIs (like YouTube IDs), they must be thoroughly sanitized if they are *also* used in local filesystem operations. The cache manager (`cache_manager.py`) correctly sanitized IDs before caching, but the main tool script failed to do so for progress logs.
**Prevention:** Always validate and sanitize user input before using it to construct file paths. Restrict IDs to an expected safe character set (e.g., alphanumeric, hyphens, and underscores) and reject or filter out anything else.
