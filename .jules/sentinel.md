## 2026-06-27 - [TOCTOU in Credential Management]
**Vulnerability:** OAuth tokens and client secrets were being saved and copied without strictly setting file permissions. Python's `shutil.copy2` and standard file operations could allow credentials to be created with overly permissive defaults (e.g. 0o644 or 0o666), enabling unauthorized access to secrets.
**Learning:** Standard library functions like `shutil.copy2` preserve source permissions which might not be secure enough, while `write_text` relies on system umask defaults. Relying on default file permissions when managing highly sensitive files (e.g. `client_secret.json`, `token.json`) is a security risk.
**Prevention:** Always enforce secure permissions when creating sensitive files by explicitly using `Path.touch(mode=0o600, exist_ok=True)` and `Path.chmod(0o600)` before writing, and ensure parent directories use `0o700`. Avoid `shutil.copy` or `shutil.copy2` in favor of `shutil.copyfile` to explicitly manage target permissions.
