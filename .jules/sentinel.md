## 2025-06-23 - Prevent TOCTOU and Enforce Secure File Permissions for Secrets
**Vulnerability:** OAuth credentials and tokens (`client_secret.json`, `token.json`) were being written and copied into directories without explicitly restricted permissions, opening the possibility for credentials leakage on shared file systems or environments.
**Learning:** Functions like `shutil.copy2` preserve source file permissions which might be insecure (e.g. world readable), and `.write_text` along with simple `mkdir()` may create secrets with permissive defaults depending on the user's `umask`.
**Prevention:** Always explicitly define permissions for secret stores (`0o700` for directories and `0o600` for files). To prevent Time-of-Check to Time-of-Use (TOCTOU) conditions, use `path.touch(mode=0o600, exist_ok=True)` combined with `.chmod(0o600)` to strictly enforce permissions before the file content is populated via `write_text` or `shutil.copyfile` (avoid `shutil.copy2` for secrets).
