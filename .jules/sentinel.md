## 2024-05-15 - [TOCTOU & Insecure Secrets Permissions]
**Vulnerability:** File handling logic for writing credentials/OAuth tokens was missing restrictive file permissions and susceptible to Time-of-Check to Time-of-Use (TOCTOU) race conditions when writing using `shutil.copy2` and `write_text` directly.
**Learning:** Python's standard file copying/writing functions do not automatically enforce secure restrictive permissions. `Path.touch(mode=0o600, exist_ok=True)` only sets permissions if the file is newly created, but if the file already exists, it will leave the insecure permissions intact. Also, `shutil.copy` copies the source file permissions, while `shutil.copyfile` does not.
**Prevention:** For files containing secrets, use `Path.touch(mode=0o600, exist_ok=True)`, but also explicitly call `.chmod(0o600)` to guarantee permissions on existing files. Use `shutil.copyfile` instead of `shutil.copy` or `shutil.copy2`, and ensure parent directories enforce `0o700` permissions.
