## 2024-07-02 - [CRITICAL] Prevent Insecure Permissions for Sensitive OAuth Credentials
**Vulnerability:** OAuth credentials (`token.json`) and their parent directory were being created with default, insecure permissions, which could potentially expose sensitive authentication tokens to unauthorized local users.
**Learning:** Default permissions when writing files (especially tokens/credentials) can be too permissive. Standard file-writing functions do not guarantee secure permissions. In addition, when updating existing files, using `write_text()` or simple assignment might retain the insecure permissions.
**Prevention:** Always enforce secure file permissions for sensitive data. Use `.mkdir(mode=0o700)` for directories containing sensitive data. Use `.touch(mode=0o600, exist_ok=True)` followed by `.chmod(0o600)` before writing credentials to ensure secure permissions and prevent Time-of-Check to Time-of-Use (TOCTOU) vulnerabilities. Use `shutil.copyfile` instead of `shutil.copy` or `shutil.copy2` when copying sensitive files to avoid inheriting insecure permissions.
