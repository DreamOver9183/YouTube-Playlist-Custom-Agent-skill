## 2025-02-28 - TOCTOU vulnerabilities and secure file permissions for secrets
**Vulnerability:** Insecure file permissions when creating or copying secret files (credentials and tokens). The code used `shutil.copy2` and `Path.write_text` which did not enforce secure permissions on the destination files, and did not secure the directories containing them.
**Learning:** `shutil.copy` and `shutil.copy2` can copy insecure permissions from the source file. Additionally, to prevent Time-of-Check to Time-of-Use (TOCTOU) vulnerabilities, `Path.touch(mode=0o600, exist_ok=True)` followed explicitly by `.chmod(0o600)` should be used to ensure the file is created with secure permissions and existing file permissions are restricted.
**Prevention:** Always use `Path.touch(mode=0o600, exist_ok=True)` followed by `.chmod(0o600)` when creating or overwriting sensitive files. Use `shutil.copyfile` instead of `shutil.copy2` to prevent copying insecure source file permissions. Enforce `0o700` permissions on directories containing secrets.
