## 2024-06-28 - Secure File Copying and File Permissions
**Vulnerability:** OAuth credentials and tokens were being created with default file permissions and potentially inheriting insecure source permissions when copying (via `shutil.copy2`). This could allow unauthorized local users to access sensitive Google API credentials.
**Learning:** `shutil.copy2` preserves the original file's metadata, including permissions. If a user downloaded their OAuth credentials file to a world-readable directory like `Downloads`, copying it with `copy2` persists those insecure permissions to the hidden `.gemini` folder. Furthermore, standard file creation does not prevent a Time-of-Check to Time-of-Use (TOCTOU) vulnerability where a file could be swapped or its permissions read before `chmod` applies.
**Prevention:** Always explicitly create directories meant for secrets with `mode=0o700`. Before writing or copying sensitive data, initialize the file safely using `Path.touch(mode=0o600, exist_ok=True)` immediately followed by `Path.chmod(0o600)` (since `touch` does not alter permissions of an existing file). Finally, use `shutil.copyfile` instead of `copy2` to ensure only the file content is copied without retaining the potentially insecure metadata from the source file.
