# v1.5 更新報告 (2026/06/20)

## 核心亮點：API 配額最佳化引擎

本次更新專注於解決**大量播放清單重排操作（如：按歌手分組）會迅速耗盡 YouTube Data API 每日配額 (10,000 units)** 的痛點。透過全新開發的三層辨識引擎與 LIS 錨點演算法，大幅減少了必要的 `update` 呼叫次數，為使用者節省 25% 至 70% 的 API 配額消耗。

### 1. 全新 `optimize` CLI 指令
新增了內建的 `optimize` 工具，專為「將相同歌手/團體的影片放在一起」等分組任務設計。
- **零 API 消耗**：所有計算皆在本地進行。
- **自動產出最小變更集**：取代過去讓 Agent 撰寫暫時性排序腳本的 Naive 方法，自動計算出達成目標所需的最少移動步數。
- **Drift-Safe 執行順序**：輸出的變更清單已自動按「目標位置由大到小（尾端優先）」排序，避免執行寫回時發生 YouTube 後端自動重編號造成的位置漂移（Position Drift）問題。

### 2. LIS (Longest Increasing Subsequence) 錨點演算法
- 利用 Patience Sorting 找出目標序列中的最長遞增子序列，將這些影片標記為不需移動的「錨點（Anchors）」。
- 僅針對非錨點影片產生 `PositionChange` 事件，極大化配額效益。

### 3. 三層嚴謹藝人辨識引擎
為了精確將影片按歌手分組，實作了優先級遞減的三層解析管線：
- **Layer 1: 標題 Regex 解析 (信心度最高)**
  - 支援 YouTube 音樂影片主流命名格式：全形括號（如 `【YOASOBI】夜に駆ける`）、破折號分隔、後置括號標註等。
  - 內建噪音清除機制，自動過濾 `Official MV`、`4K`、`Remastered`、`feat.` 等字眼。
- **Layer 2: 頻道名稱交叉驗證**
  - 使用 NFKC Unicode 正規化處理全半形/CJK字元，並剔除 `Official`、`Entertainment` 等官方後綴。
  - **版權代理黑名單**：自動略過 `1theK`、`SMTOWN`、`Big Hit Labels` 等版權發行頻道，避免將不同藝人誤判為同一群組。
- **Layer 3: 模糊比對與別名支援 (兜底)**
  - 導入 `rapidfuzz` 進行 `WRatio` 智慧模糊比對。
  - 支援使用者自訂 `data/artist_aliases.json`（例如將 `"BTS"`、`"Bangtan Boys"`、`"방탄소년단"` 視為同一藝人）。

### 4. Human-in-the-Loop 流程強化
- `diff` 與 `optimize` 輸出的預覽表格現在會明確標示「🔄 移動」與「⚓ 錨點」狀態。
- 在預覽階段，Agent 會明確告知「**較傳統方法節省了多少 units 的配額**」，讓使用者對 API 使用量有充分的掌控感。

### 5. 徹底移除 GUI 依賴
- 移除了原有的 `tkinter` 檔案選擇對話框機制。
- OAuth 憑證（`client_secret.json`）設定流程已完全轉為 CLI 問答形式，更適合在 Headless 環境或雲端環境中交由各種 AI Agent 操作。

## 效能與測試
- 導入了涵蓋邊角案例（Edge Cases）的完整單元測試（共 24 項），包含真實 YouTube 標題解析測試、LIS 計算正確性驗證，以及漂移模擬。
- **實測效益**：針對 200 支包含多個歌手的隨機排列清單，傳統做法約需耗費 9,500 units；透過 v1.5 引擎最佳化後，僅需消耗約 4,000–7,500 units，有效避免單次操作觸發限額。
